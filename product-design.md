# 投稿代理（Submission Agent）产品设计文档

> 版本：v1.1 | 日期：2026-02-21（v1.1 更新：AI能力边界分层策略）
> 使命：帮助中国创作者将作品投稿到国际竞赛，打破信息差和语言壁垒

---

## 一、市场分析

### 1.1 痛点定义

中国创作者参与国际竞赛面临三重壁垒：

| 壁垒 | 具体表现 | 影响程度 |
|------|----------|----------|
| 信息差 | 不知道有哪些竞赛、截止日期、投稿要求 | 🔴 致命 |
| 语言壁垒 | 投稿页面全英文、作品需英文翻译、Bio/Synopsis 撰写 | 🔴 致命 |
| 流程壁垒 | 注册平台、格式要求、支付方式（PayPal/信用卡）、同时投稿规则 | 🟡 高 |

### 1.2 目标用户画像

**核心用户：有作品、想出海、但被流程卡住的中国创作者**

| 用户类型 | 特征 | 付费意愿 | 规模估算 |
|----------|------|----------|----------|
| 文学创作者 | 写小说/诗歌/散文，活跃在豆瓣/微信公众号/小红书 | 中 | 50万+ |
| 摄影师 | 有作品集，想参加国际摄影奖 | 高 | 100万+ |
| 设计师/插画师 | 独立设计师、自由插画师，想拿国际奖项背书 | 高 | 80万+ |
| 编剧/影视创作者 | 有剧本或短片，想投国际电影节 | 高 | 20万+ |
| 学生 | 想投国际论文/设计竞赛丰富简历 | 低-中 | 200万+ |

**典型用户故事：**
> 小林，28岁，自由摄影师。在小红书有3万粉丝，拍了一组很满意的黑白街拍。听说过 Sony World Photography Awards，但打开官网全是英文，不知道怎么投，也不确定自己够不够格。最后放弃了。

### 1.3 市场规模估算

- 中国创意产业从业者：约 2,200 万人（2024 国家统计局）
- 有国际投稿意愿的比例（保守估计）：1-3%
- 潜在用户池：22-66 万人
- 年均客单价（按进阶版 ¥399 计算）：¥399
- 可触达市场（TAM）：¥8,800万 - ¥2.6亿
- 初期可服务市场（SOM，第一年）：500-2,000 用户 → ¥20万-¥80万

## 二、竞品分析

### 2.1 国际竞品

| 平台 | 定位 | 核心功能 | 定价 | 优势 | 劣势（对中国用户） |

### 2.2 国内竞品/相关服务

| 服务 | 定位 | 与我们的关系 |
|------|------|-------------|
| **LetPub** | SCI 论文润色+投稿推荐 | 学术领域的成功先例，证明"翻译+代投"模式可行，但不覆盖创意领域 |

| **留学中介** | 申请代理 | 模式类似但面向不同场景，定价远高于我们 |

### 2.3 竞争定位

```
                    高服务深度（代投+翻译+追踪）
                           ↑
                           |
                    【投稿代理】← 我们在这里
                           |
        LetPub(学术) ←     |
                           |
    低覆盖度 ←————————————+————————————→ 高覆盖度
                           |
              Duotrope →   |    ← Submittable
                           |
         Grinder →         |    ← FilmFreeway
                           |
                           ↓
                    低服务深度（仅搜索/仅表单）
```

**核心差异化：**
1. 中文界面 + 中文服务 → 唯一面向中国创作者的国际竞赛投稿平台
2. AI 翻译+人工润色 → 解决语言壁垒（竞品都不做翻译）
3. 智能匹配 → 不只是搜索，而是根据作品属性推荐最佳竞赛
4. 全流程代投 → 从匹配到投递到追踪，一站式服务

## 三、产品架构

### 3.1 核心功能模块

```
┌─────────────────────────────────────────────────┐
│                  投稿代理平台                      │
├──────────┬──────────┬──────────┬────────┬────────┤
│ 竞赛数据库 │ 智能匹配  │ AI翻译润色 │ 格式适配 │ 投稿追踪 │
│          │          │          │        │        │
│ 58+竞赛   │ 作品分析   │ 中→英翻译  │ 自动排版 │ 状态看板 │
│ 分类索引   │ 多维评分   │ 文学润色   │ 字数适配 │ 截止提醒 │
│ 截止日历   │ 预算规划   │ Bio撰写   │ 文件命名 │ 结果通知 │
│ 定期更新   │ 投稿策略   │ Synopsis  │ 格式转换 │ 历史记录 │
└──────────┴──────────┴──────────┴────────┴────────┘
```

### 3.2 功能详述

#### 模块一：竞赛数据库

**现有基础：** 58 条竞赛数据（2026年2月17日更新），覆盖文学、摄影、设计、插画、影视五大类。

**数据结构（每条竞赛）：**
- 基本信息：名称、领域、子类别、官网
- 投稿要求：语言、字数范围、主题偏好、是否接受已发表作品
- 费用与奖金：投稿费（USD）、奖金金额、奖品描述
- 时间线：截止日期、结果公布日期
- 资格限制：国籍、年龄、经验等级
- 声望评分：1-10 人工标注
- 投稿平台：Submittable / 官网 / FilmFreeway 等

**更新机制：**
- 每月一次全量扫描（AI agent 自动搜索 + 人工审核）
- 截止日期前 30/14/7 天自动提醒
- 用户可提交新竞赛信息（社区贡献）

#### 模块二：智能匹配

**已有框架：** 完整的匹配算法伪代码（matching-framework.md），包含硬性条件过滤 + 软性评分系统。

**匹配维度与权重：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 类型匹配 | 20% | 作品类型 vs 竞赛类别 |
| 主题匹配 | 20% | 作品主题 vs 竞赛偏好 |
| 风格匹配 | 15% | 文学/商业/实验/传统 |
| 经验等级 | 10% | 业余/新锐/专业 |
| 字数优化 | 10% | 与竞赛要求的契合度 |
| 性价比 | 10% | 投稿费 vs 奖金 ROI |
| 声望 | 10% | 竞赛知名度和含金量 |
| 时间充裕度 | 5% | 距截止日期的天数 |

**输出：** 匹配度 0-100 分 + 推荐投稿计划（含预算控制）

#### 模块三：AI 翻译 + 润色

这是核心付费功能，也是最大的技术壁垒。

**翻译流程：**
```
中文原稿 → AI 初译（Claude/GPT） → 文学风格适配 → 人工审校（可选） → 定稿
```

**翻译层级：**

| 层级 | 方式 | 适用场景 | 定价 |
|------|------|----------|------|
| 基础翻译 | AI 直译 + 基础润色 | 摄影/设计类的描述文字、Bio | ¥0.1/字 |
| 文学翻译 | AI 翻译 + 风格适配 + 语境优化 | 短篇小说、散文、诗歌 | ¥0.3/字 |
| 专业翻译 | AI 翻译 + 人工母语审校 | 重要竞赛的核心作品 | ¥0.8/字 |

**附加服务：**
- 作者简介（Bio）撰写：¥49/篇
- 作品简介（Synopsis）撰写：¥99/篇
- Cover Letter 撰写：¥79/篇

#### 模块四：格式适配

不同竞赛对投稿格式有不同要求，这是很多创作者放弃的原因之一。

**自动处理：**
- 文件格式转换（.docx / .pdf / .rtf）
- 页面设置（字体、字号、行距、页边距）
- 标题页生成（含/不含作者信息，视竞赛要求）
- 字数统计与适配建议
- 文件命名规范（LastName_Title_Category.docx）
- 图片规格调整（摄影类：DPI、尺寸、色彩空间）

#### 模块五：投稿追踪

**追踪看板：**
```
┌──────────────────────────────────────────────┐
│  我的投稿                                      │
├──────────┬──────────┬──────────┬──────────────┤
│ 待投递(3) │ 已投递(5) │ 审核中(2) │ 已出结果(1)  │
├──────────┴──────────┴──────────┴──────────────┤
│ 📝 Bridport Prize 2026                        │
│    作品：《铭刻》英译版                          │
│    状态：已投递 ✅ | 投递日期：2026-03-15        │
│    截止：2026-05-31 | 结果预计：2026-09          │
│    投稿费：£15 | 奖金：£5,000                   │
│    [查看确认截图] [查看投稿详情]                  │
└──────────────────────────────────────────────┘
```

**通知机制：**
- 截止日期提醒（30/14/7/3/1 天）
- 投递确认通知（含截图）
- 结果公布提醒
- 新竞赛推荐（基于用户画像）

## 五、技术方案

### 5.1 MVP 技术栈

**原则：最小可行，快速验证，不过度工程化。**

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端 | 静态页面（已有 landing-page.html） + 微信小程序 | 中国用户习惯，开发成本低 |
| 后端 | OpenClaw Agent（现有基础设施） | 零额外成本，AI 原生 |
| 数据库 | Markdown 文件 → SQLite（后期） | 58 条数据用文件足够 |
| 翻译引擎 | Claude API / GPT API | 已有 MMKG 天卡，成本极低 |
| 匹配算法 | Python 脚本（已有框架） | 直接实现 matching-framework.md |
| 支付 | 微信支付 / 支付宝 | 国内用户标配 |
| 通知 | 微信服务号 / 小红书私信 | 触达率高 |

### 5.2 AI 翻译技术细节

**文学翻译 Prompt 策略：**
```
角色：你是一位精通中英文学翻译的译者，风格参考葛浩文（Howard Goldblatt）。
任务：将以下中文{类型}翻译为英文，保留原作的文学性和情感张力。
要求：
1. 不是逐字翻译，而是传达原文的意境和节奏
2. 使用自然的英语表达，避免翻译腔
3. 保留文化特色元素，必要时加注释
4. 输出格式符合{竞赛名称}的投稿要求
```

**质量保障：**
- 双模型交叉翻译（Claude + GPT），取最优
- 回译验证（英→中回译，对比原文偏差）
- 可选人工母语审校（远期接入 Fiverr/Upwork 译者）

## 5.5、AI 能力边界与服务分层策略（v1.1 新增）

### 核心认知：AI 目前的审美判断力不足以评估视觉作品

当前 AI 模型（包括多模态模型）在文本理解和生成方面已经成熟，但在视觉审美判断方面仍有明显短板：
- 无法可靠判断一张照片是"好"还是"伟大"
- 无法评估插画的构图、色彩、情感张力是否达到竞赛水准
- 无法判断设计作品是否具有"Red Dot 级别"的创新性

**这直接决定了我们的服务边界。**

### 按创作类型的服务深度分层

| 服务能力 | 文本创作（小说/诗歌/散文/剧本） | 视觉创作（摄影/设计/插画） |
|----------|-------------------------------|--------------------------|
| 竞赛信息推送 | ✅ 全能力 | ✅ 全能力 |
| 智能匹配推荐 | ✅ 深度（风格+评委偏好+获奖概率） | ⚠️ 浅层（仅基于类别/主题/资格） |
| 作品质量评估 | ✅ 能做（语言、结构、风格） | ❌ 做不了（审美判断不可靠） |
| 翻译/润色 | ✅ 核心价值 | N/A |
| Bio/Synopsis/Cover Letter | ✅ 能做 | ✅ 能做 |
| 格式适配 | ✅ 能做 | ✅ 能做（DPI/尺寸/色彩空间等技术层面） |
| 投稿流程指导 | ✅ 能做 | ✅ 能做 |
| 选稿建议（投哪篇/哪张） | ✅ 能做 | ❌ 做不了（需要人工审美） |
| 获奖概率评估 | ✅ 多维模型 | ⚠️ 仅基于竞争密度等客观指标 |

### 战略结论：文学类先行，视觉类跟进

**Phase 0-2：聚焦文本创作的全链条代理服务**
- 文学类（闪小说/短篇/诗歌/散文/剧本）是我们服务价值最大的赛道
- 翻译+润色+风格匹配+获奖概率评估，全链条 AI 都能做
- 通过文学类代理积累客户源和口碑

**Phase 2+：视觉类提供信息服务层**
- 摄影/设计/插画只做信息推送+匹配+格式适配+流程指导
- 不碰"帮你选哪张照片投"这种需要审美判断的事
- 等多模态模型审美能力成熟，或建立人工审稿团队后，再扩展到全流程

**Phase 3+：引入人工审美环节（远期）**
- 与专业策展人/编辑合作，提供视觉作品选稿服务
- 或等 AI 审美能力突破后，重新评估

### 对之前分析的修正

之前说"摄影是中国创作者最佳切入点（无语言壁垒）"——这对用户来说是对的，但对我们的服务来说不是。我们能提供的服务深度：

- 文学类：全链条（翻译→润色→匹配→格式→指导）→ **高附加值**
- 视觉类：信息层（推送→匹配→格式→指导）→ **低附加值**

**用户获客策略：先用文学类的深度服务建立口碑，再用信息服务覆盖视觉类创作者。**

## 七、风险分析与应对

### 7.1 核心风险

| 风险 | 严重度 | 概率 | 应对策略 |
|------|--------|------|----------|
| **翻译质量不达标** | 🔴 高 | 中 | 双模型交叉翻译 + 回译验证 + 可选人工审校；初期只接摄影/设计类（无语言要求），逐步扩展到文学类 |
| **获客成本过高** | 🔴 高 | 中 | 小红书内容营销为主（零成本）；种子用户口碑传播；与写作社区/豆瓣小组合作 |
| **目标用户太小众** | 🟡 中 | 中 | 先从摄影/设计切入（用户基数大、无语言壁垒）；文学类作为高端服务 |
| **版权纠纷** | 🟡 中 | 低 | 明确服务协议：代投不等于代理版权；所有版权归创作者；投稿前获得书面授权 |
| **竞赛数据过时** | 🟡 中 | 中 | 月度自动更新 + 用户反馈机制 + 投稿前人工确认 |
| **竞品模仿** | 🟢 低 | 中 | 先发优势 + 数据积累 + 社区壁垒；核心壁垒是服务质量而非技术 |
| **支付壁垒** | 🟡 中 | 中 | 部分竞赛需 PayPal/国际信用卡，国内用户可能没有；提供代付服务（加收手续费） |

### 7.2 交叉评审中指出的风险回应

来源：CROSS-REVIEW.md 投稿代理部分

| 评审指出的风险 | 我们的回应 |
|---------------|-----------|
| 翻译质量是核心瓶颈 | Phase 0 先做摄影/设计类（无翻译需求），验证代投流程；文学翻译用 AI+人工双保险 |
| 版权纠纷 | 服务协议明确版权归属，代投前获取书面授权，不涉及版权转让 |
| 获客成本高 | 内容营销为主，零广告预算；利用竞赛信息本身作为引流内容（干货帖自带传播力） |
